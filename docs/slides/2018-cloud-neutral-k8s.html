Cloud Agnostic StudioML - 2018 - 2019

Motivation

Ever growing list of deployment configurations and business interests

  • StudioML is totally worthwhile and useful
  • StudioML has multi-cloud capability now


Downside

  • linear development cost for each deployment config
  • testing cost/complexity slope
  • not so adaptable/flexible to business changes

One proposal for StudioML Sentient effort

  • Encourage StudioML to gain increased independence, community governance for pypi for example
  • Fix https://github.com/studioml/studio/community
  • Simplify hosting for StudioML docs etc


Decouple from technology choices made by/for StudioML stakeholders


  • provide one broadly applicable portable deployment/technology stack
  • encourage the StudioML community to deal with use cases for individual stakeholders
  • keep future Sentient StudioML work to being cloud/deployment model agnostic
